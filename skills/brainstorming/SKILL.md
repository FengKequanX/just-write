# 内容选题讨论

把模糊的写作想法变成明确的选题。通过 2-3 个问题帮用户理清主题、角度和读者，然后输出选题检查点，等待确认。

**本 skill 只做选题讨论，不做写作、不做设计文档、不调用其他 skill。** 输出检查点后回到 just-write 主流程。

---

## 流程（最多 3 步）

### 1. 了解输入

- 用户输入明确（如"写一篇关于 Altman 改口的文章"）→ 跳过提问，直接到第 3 步
- 用户输入模糊（如"有没有适合写的"）→ 进入第 2 步

### 2. 搜索热点 + 提出方向

如果用户没有明确主题：

1. 调用 `WebSearch` 或 aihot skill 搜索近期 AI/科技热点
2. 提出 2-3 个选题方向，每个方向包含：
   - 主题 + 为什么现在写（时效性）
   - 可能的角度（痛点/反常识/个人体验）
3. 用户选择后，一次问一个问题（最多 3 个）：
   - 核心观点是什么？
   - 目标读者是谁？
   - 有独特的个人视角或经历吗？

**一次只问一个问题。** 不要连续抛出多个问题。

### 3. 输出检查点

输出选题确认检查点，等待用户说"确认选题"：

```
【选题确认】
- 文章主题：...
- 核心观点：...
- 目标读者：...
- 时效性/切入点：...

确认后回复"确认选题"。
```

**检查点输出后必须停下来等用户确认。** 不能自作主张进入写作。

---

## 与 just-write 工作流的衔接

本 skill 是 just-write 工作流的 Step 1。输出检查点并收到用户"确认选题"后，just-write 主流程会推进到 Step 2（内容生成）。

不要：
- 调用 writing-plans、frontend-design 等其他 skill
- 写设计文档
- 进入代码相关讨论
- 输出超过上述格式的检查点
